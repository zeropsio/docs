title: 'Choose how finished work ships'
description: 'Delivery preference is now covered in Build with ZCP.'
---

Delivery preference is part of the normal ZCP development lifecycle. The current guide lives in [Build with ZCP → Choose delivery preference](/zcp/workflows/build-with-zcp#choose-delivery-after-proof).

That section explains the three user-facing choices: keep direct deploy, push verified work to git, or hand off to CI/release tooling. ZCP records the delivery choice so later work can follow the same path without re-explaining the handoff every time.

For exact internal labels, use [Workflow terms](/zcp/reference/agent-workflow#delivery-modes).
