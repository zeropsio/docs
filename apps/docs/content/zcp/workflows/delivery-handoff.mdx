This page moved to [Build with ZCP](/zcp/workflows/build-with-zcp).

Delivery preference is part of the normal ZCP development lifecycle. The current guide explains the three user-facing choices: keep direct deploy, push verified work to git, or hand off to CI/release tooling. ZCP records the delivery choice so later work can follow the same path without re-explaining the handoff every time.

For exact internal labels, use [Workflow terms](/zcp/reference/agent-workflow#delivery-modes).
