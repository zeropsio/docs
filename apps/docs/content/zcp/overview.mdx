Zerops Control Plane (ZCP) lets coding agents operate a real Zerops project through scoped tools, live state, deploys, logs, and verification. Through ZCP, an agent does real application work — not just suggesting commands you run elsewhere.

:::caution Public preview
Zerops Control Plane (ZCP) is in public preview. Use it for development and staging projects, keep production in a separate Zerops project, and expect setup details to change.
:::

## What ZCP is

ZCP runs as a service inside your Zerops project, or as a binary on your laptop. Either way, it exposes the same project to a coding agent — the same services, env vars, deploy pipeline, logs, and verify loop your app actually uses. The agent operates against the platform; it does not work in a sandbox.

What ZCP gives an agent:

- **Project awareness** — services, runtime shape, env vars, recent activity
- **Service targeting** — distinguishing the application runtime from managed dependencies
- **A real deploy and verify loop** — proving the app works on the selected runtime, not stopping after writing code
- **Delivery handoff** — explaining whether future changes stay on direct deploy, move through git, or need a manual or external step

You hold the product intent: what to build, which target matters, what counts as done. ZCP fills in the Zerops-specific path and explains the choices it made.

## Who it is for

ZCP is for developers who use a coding agent and want it to do real application work — provision services, edit code, deploy through the Zerops pipeline, verify against real URLs, recover from failure, and report back. You hold the product intent and quality bar; the agent handles the platform-specific path; ZCP is the bridge between them.

Today, ZCP supports **Claude Code**, paired with a **Claude Pro or Max subscription**. Other MCP-aware agents will follow as their integrations stabilize. You bring the agent and the subscription; ZCP is the control plane in the middle.

> The LLM does the engineering. The developer holds intent, quality, and decisions. ZCP holds reality.

## What stays outside ZCP

ZCP is a control plane, not the platform itself. The Zerops platform — the [build and deploy pipeline](/features/pipeline), [permissions](/features/rbac), [deployment lifecycle](/guides/deployment-lifecycle), networking, scaling — keeps its own canonical documentation. ZCP docs explain how an agent uses these primitives; they do not replace the platform reference.

Recipes are Zerops-published starters. ZCP users select recipes; this section does not cover authoring or publishing them.
