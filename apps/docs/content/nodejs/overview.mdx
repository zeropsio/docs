[Node.js ↗](https://nodejs.org/en) is an asynchronous event-driven JavaScript runtime designed to build scalable network applications.

## 1. Try a Zerops Recipe

A **recipe** is a minimal working app in a public GitHub repository, already wired up with a `zerops.yml` — no setup needed on your part, just deploy and see it run.

<CustomCard title="Deploy the Node.js recipe" emoji="🚀">
  A bare-bones Node.js app with a single endpoint, connected to a PostgreSQL database. One click, no changes needed.

  <DeployButton link="node-js" badgeWidth="200" />
</CustomCard>

<Dropdown>
<DropdownItem title="Import manually or fork the recipe">

Or import manually — log in to [Zerops GUI ↗](https://app.zerops.io), click **Import a project** in the Projects box, and paste this YAML ([source ↗](https://github.com/zeropsio/recipe-nodejs/blob/main/zerops-project-import.yaml)):

```yaml
project:
  name: recipe-nodejs
  # location: eu-central   # eu-central (EU, Prague, default) or us-east-1 (US East, New York)
  tags:
    - zerops-recipe

services:
  - hostname: api
    type: nodejs@20
    enableSubdomainAccess: true
    buildFromGit: https://github.com/zeropsio/recipe-nodejs

  - hostname: db
    type: postgresql@16
    mode: NON_HA
    priority: 1
```

Click **Import project** and wait for all pipelines to finish. The subdomain URL in the **IP Addresses & Public Routing** box should return `Hello, World!`.

To fork and build on top: [fork the repository ↗](https://github.com/zeropsio/recipe-nodejs), replace `buildFromGit` with your fork's URL, then push — Zerops rebuilds on every push.

</DropdownItem>
</Dropdown>

## 2. Deploy your own project

Already have a Node.js repo? Add a `zerops.yml` file to its root, then [create a Node.js service](/nodejs/how-to/create) in Zerops and point it at your repository.

```yaml
zerops:
  - setup: app         # matches your service hostname in Zerops
    build:
      base: nodejs@latest
      buildCommands:
        - npm i
        - npm run build
      deployFiles:
        - dist
        - package.json
        - node_modules
      cache: node_modules
    run:
      base: nodejs@latest
      ports:
        - port: 3000
          httpSupport: true
      start: npm start
```

See the [full `zerops.yml` reference](/nodejs/how-to/build-pipeline) for all available options.
