[Go ↗](https://go.dev/) is a statically typed, compiled high-level programming language designed at Google.

## 1. Try a Zerops Recipe

A **recipe** is a minimal working app in a public GitHub repository, already wired up with a `zerops.yml` — no setup needed on your part, just deploy and see it run.

<CustomCard title="Deploy the Go recipe" emoji="🚀">
  A bare-bones Go app with a single endpoint, connected to a PostgreSQL database. One click, no changes needed.

  <DeployButton link="golang" badgeWidth="200" />
</CustomCard>

<Dropdown>
<DropdownItem title="Import manually or fork the recipe">

Or import manually — log in to [Zerops GUI ↗](https://app.zerops.io), click **Import a project** in the Projects box, and paste this YAML ([source ↗](https://github.com/zeropsio/recipe-go-hello-world/blob/main/import-project/description.yaml)):

```yaml
project:
  name: my-first-project
  # location: eu-central   # eu-central (EU, Prague, default) or us-east-1 (US East, New York)
services:
  - hostname: helloworld
    type: go@latest
    minContainers: 1
    maxContainers: 3
    buildFromGit: https://github.com/zeropsio/recipe-go-hello-world@main
    enableSubdomainAccess: true
```

Click **Import project** and wait for all pipelines to finish. The subdomain URL in the **IP Addresses & Public Routing** box should return `Hello, World!`.

To fork and build on top: [fork the repository ↗](https://github.com/zeropsio/recipe-go-hello-world), replace `buildFromGit` with your fork's URL, then push — Zerops rebuilds on every push.

</DropdownItem>
</Dropdown>

## 2. Deploy your own project

Already have a Go repo? Add a `zerops.yml` file to its root, then [create a Go service](/go/how-to/create) in Zerops and point it at your repository.

```yaml
zerops:
  - setup: app         # matches your service hostname in Zerops
    build:
      base: go@latest
      buildCommands:
        - go build -o app
      deployFiles:
        - app
    run:
      base: go@latest
      ports:
        - port: 8080
          httpSupport: true
      start: ./app
```

See the [full `zerops.yml` reference](/go/how-to/build-pipeline) for all available options.
