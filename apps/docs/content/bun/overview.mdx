[Bun ↗](https://bun.sh/) is an all-in-one JavaScript runtime and toolkit designed for speed.

## 1. Try a Zerops Recipe

A **recipe** is a minimal working app in a public GitHub repository, already wired up with a `zerops.yml` — no setup needed on your part, just deploy and see it run.

<CustomCard title="Deploy the Bun recipe" emoji="🚀">
  A bare-bones Bun app with a single endpoint, connected to a PostgreSQL database. One click, no changes needed.

  <DeployButton link="bun" badgeWidth="200" />
</CustomCard>

<Dropdown>
<DropdownItem title="Import manually or fork the recipe">

Or import manually — log in to [Zerops GUI ↗](https://app.zerops.io), click **Import a project** in the Projects box, and paste this YAML ([source ↗](https://github.com/zeropsio/recipe-bun/blob/main/zerops-project-import.yaml)):

```yaml
project:
  name: recipe-bun
  # location: eu-central   # eu-central (EU, Prague, default) or us-east-1 (US East, New York)
  tags:
    - zerops-recipe

services:
  - hostname: api
    type: bun@1.1
    enableSubdomainAccess: true
    buildFromGit: https://github.com/zeropsio/recipe-bun

  - hostname: db
    type: postgresql@16
    mode: NON_HA
    priority: 1
```

Click **Import project** and wait for all pipelines to finish. The subdomain URL in the **IP Addresses & Public Routing** box should return `Hello, World!`.

To fork and build on top: [fork the repository ↗](https://github.com/zeropsio/recipe-bun), replace `buildFromGit` with your fork's URL, then push — Zerops rebuilds on every push.

</DropdownItem>
</Dropdown>

## 2. Deploy your own project

Already have a Bun repo? Add a `zerops.yml` file to its root, then [create a Bun service](/bun/how-to/create) in Zerops and point it at your repository.

```yaml
zerops:
  - setup: app         # matches your service hostname in Zerops
    build:
      base: bun@latest
      buildCommands:
        - bun i
        - bun run build
      deployFiles:
        - dist
        - package.json
        - node_modules
      cache: node_modules
    run:
      base: bun@latest
      ports:
        - port: 3000
          httpSupport: true
      start: bun start
```

See the [full `zerops.yml` reference](/bun/how-to/build-pipeline) for all available options.
