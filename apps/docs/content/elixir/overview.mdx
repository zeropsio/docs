[Elixir ↗](https://elixir-lang.org/) is a dynamic, functional language for building scalable and maintainable applications on the Erlang VM.

## 1. Try a Zerops Recipe

A **recipe** is a minimal working app in a public GitHub repository, already wired up with a `zerops.yml` — no setup needed on your part, just deploy and see it run.

<CustomCard title="Deploy the Elixir recipe" emoji="🚀">
  A bare-bones Elixir app with a single endpoint, connected to a PostgreSQL database. One click, no changes needed.

  <DeployButton link="elixir" badgeWidth="200" />
</CustomCard>

<Dropdown>
<DropdownItem title="Import manually or fork the recipe">

Or import manually — log in to [Zerops GUI ↗](https://app.zerops.io), click **Import a project** in the Projects box, and paste this YAML ([source ↗](https://github.com/zeropsio/recipe-elixir/blob/main/zerops-project-import.yaml)):

```yaml
project:
  name: recipe-elixir
  # location: eu-central   # eu-central (EU, Prague, default) or us-east-1 (US East, New York)
  tags:
    - zerops-recipe

services:
  - hostname: api
    type: elixir@1.16
    enableSubdomainAccess: true
    buildFromGit: https://github.com/zeropsio/recipe-elixir

  - hostname: db
    type: postgresql@16
    mode: NON_HA
    priority: 1
```

Click **Import project** and wait for all pipelines to finish. The subdomain URL in the **IP Addresses & Public Routing** box should return `Hello, World!`.

To fork and build on top: [fork the repository ↗](https://github.com/zeropsio/recipe-elixir), replace `buildFromGit` with your fork's URL, then push — Zerops rebuilds on every push.

</DropdownItem>
</Dropdown>

## 2. Deploy your own project

Already have an Elixir repo? Add a `zerops.yml` file to its root, then [create an Elixir service](/elixir/how-to/create) in Zerops and point it at your repository.

```yaml
zerops:
  - setup: app         # matches your service hostname in Zerops
    build:
      base: elixir@latest
      buildCommands:
        - mix deps.get --only prod
        - mix compile
        - mix release
      deployFiles:
        - _build/prod/rel/app/
    run:
      base: elixir@latest
      ports:
        - port: 4000
          httpSupport: true
      start: _build/prod/rel/app/bin/app start
```

See the [full `zerops.yml` reference](/elixir/how-to/build-pipeline) for all available options.
